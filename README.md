# Naves Scraper

Automated scraper for industrial warehouse listings (naves industriales) from [MilAnuncios.com](https://www.milanuncios.com), with a full-stack dashboard and Webflow CMS integration.

## Architecture

```
save_session.py -> session.json -> scraper_engine.py
  -> integrations/milanuncios.py (zendriver headful Chrome)
  -> integrations/parser.py (window.__INITIAL_PROPS__ JSON)
  -> db.py (SQLite WAL, INSERT OR IGNORE)
  -> integrations/webflow_sync.py -> Webflow CMS API
```

**Services:**

| Service | Port | Description |
|---------|------|-------------|
| FastAPI API | :8000 | REST endpoints for scraper control, listings, scheduling, Webflow sync |
| Next.js Dashboard | :3000 | Web UI for monitoring and managing the scraper |
| websockify (VPS only) | :6080 | WebSocket bridge for remote Chrome VNC panel |

## Quick Start

```bash
bash install.sh                                # Install everything (Python, Node, deps, .env)
nano .env                                      # Fill in WEBFLOW_TOKEN, WEBFLOW_COLLECTION_ID, DASHBOARD_PASSWORD
source venv/bin/activate && python save_session.py  # Manual login to MilAnuncios (one time)
bash start.sh                                  # Start API + Dashboard in background
```

Dashboard: **http://localhost:3000** (login with `DASHBOARD_PASSWORD` from `.env`)

> For a step-by-step interactive guide (in Spanish), open `docs/setup-guide.html` in your browser.

## Prerequisites

- Python 3.11+
- Node.js 20+
- macOS 12+ or Ubuntu 22+
- Webflow API token + Collection ID
- MilAnuncios account

On Linux, `install.sh` also installs: `xvfb`, `x11vnc`, `websockify`, `fluxbox` (for headless Chrome + remote VNC panel).

## Environment Variables

Edit `.env` after running `install.sh`:

| Variable | Required | Description |
|----------|----------|-------------|
| `WEBFLOW_TOKEN` | Yes | Webflow API token (Settings > Integrations) |
| `WEBFLOW_COLLECTION_ID` | Yes | Target Webflow CMS collection ID |
| `DASHBOARD_PASSWORD` | Yes | Password to access the web dashboard |
| `API_SECRET_KEY` | Auto | Generated by `install.sh`. Used for API authentication |
| `MAX_PAGES` | No | Pages to scrape per run. `0` = unlimited (default) |
| `MIN_SURFACE_M2` | No | Minimum surface filter in m2. Default: `1000` |
| `DOWNLOAD_IMAGES` | No | Download listing images locally. Default: `true` |
| `IMAGES_DIR` | No | Image storage directory. Default: `images` |
| `DB_PATH` | No | SQLite database path. Default: `naves.db` |
| `PROXY_SERVER` | No | HTTP proxy for Chrome. Format: `http://user:pass@host:port` |
| `CORS_ORIGINS` | No | Allowed CORS origins, comma-separated. Default: `http://localhost:3000` |
| `API_BASE_URL` | No | API URL for internal use. Default: `http://localhost:8000` |

## Service Management

```bash
bash start.sh      # Start API + Dashboard (background, survives terminal close)
bash stop.sh       # Stop all services
bash restart.sh    # Stop + Start
```

Logs are written to `logs/api.log` and `logs/frontend.log`.

## Scraper CLI

```bash
source venv/bin/activate

python scraper_engine.py                        # Incremental from last checkpoint
python scraper_engine.py --pages 5              # Limit to 5 pages
python scraper_engine.py --batch 50             # Stop after 50 new listings
python scraper_engine.py --pages 1 --dry-run    # Test mode (no DB writes)
python scraper_engine.py --reset                # Ignore checkpoint, start from page 1
```

## API Endpoints

All endpoints (except `/health`) require the `x-api-key` header.

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check |
| POST | `/api/auth/login` | Password login, returns API key |
| POST | `/api/scraper/run` | Start scraper (max_pages, dry_run, reset) |
| GET | `/api/scraper/status` | Current scraper state and progress |
| POST | `/api/scraper/stop` | Stop running scraper |
| GET | `/api/listings` | Paginated listings with filters (province, surface, price) |
| GET | `/api/logs` | Recent log lines |
| GET | `/api/cron` | Current cron schedule + next run |
| PUT | `/api/cron` | Update cron schedule |
| POST | `/api/session/renew` | Launch manual login (opens Chrome) |
| POST | `/api/session/stop` | Cancel session renewal |
| GET | `/api/session/status` | Session renewal progress |
| POST | `/api/webflow/sync` | Trigger Webflow CMS sync |
| GET | `/api/webflow/status` | Sync statistics |
| GET | `/api/vnc/status` | VNC panel availability |

| Page | Description |
|------|-------------|

## Remote Chrome Panel (VPS)

On Linux/VPS, Chrome runs inside a virtual display (Xvfb :99). The VNC stack lets you interact with Chrome from the dashboard to solve captchas or complete manual login.

Stack: `Xvfb :99` -> `x11vnc :5900` -> `websockify :6080` -> `react-vnc` in dashboard

Install dependencies (if not already installed by `install.sh`):

```bash
sudo apt-get install -y x11vnc novnc websockify
bash restart.sh
```

On macOS, Chrome opens on your display directly. No VNC needed.

## Important Rules

- **Never delete `chrome_profile/`** -- it accumulates anti-bot trust over time
- **Workers must be 1** -- `uvicorn --workers 1` (singleton Chrome instance)
- **DISPLAY=:99** on Linux (Xvfb); real display on macOS

## Session Renewal

MilAnuncios session cookies expire after ~30 minutes of inactivity. When expired:

1. The scraper stops automatically and reports `SessionExpiredException`
2. The dashboard shows a red alert with a "Renew session" button
3. Click the button (or run `python save_session.py` manually)
4. Log in to MilAnuncios in the Chrome window
5. The script auto-detects login and saves cookies
