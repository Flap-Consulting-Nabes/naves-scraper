Naves Scraper / Setup

Guía de instalación

Configura el scraper de naves industriales en menos de 10 minutos. Al finalizar tendrás el dashboard activo y el cron diario funcionando.

laptop_macmacOS 12+
terminalUbuntu 22+
codePython 3.11+
Antes de empezar
  • check_circle Cuenta en Webflow con acceso a la API — necesitarás el token y el Collection ID
  • check_circle Cuenta en MilAnuncios — para el login manual de sesión (solo la primera vez)
  • check_circle Conexión a internet activa — el instalador descarga todo lo necesario
Instalación completa — 4 pasos
1
Instalar git y Python
Elige tu sistema operativo
macOS — Homebrew + git + Python 3.12 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install git python@3.12
Ubuntu / Debian 
sudo apt update && sudo apt install -y git python3.12 python3.12-venv
2
Clonar el repositorio e instalar
El instalador configura todo: venv, dependencias y .env
terminal 
git clone https://github.com/AlejandroTechFlap/naves-scraper.git naves-scraper
cd naves-scraper
bash install.sh
Al finalizar, el script pregunta si quieres arrancar los servicios ahora. Responde N y continúa con los pasos siguientes.
3
Configurar credenciales y sesión
Edita .env con tus claves y guarda las cookies de MilAnuncios
warning .env contiene secretos — nunca lo subas a Git. Ya está en .gitignore.
paso 3a — editar credenciales 
nano .env
VariableTipoDescripción
WEBFLOW_TOKEN Requerida Token de API de Webflow (perfil → API Access)
WEBFLOW_COLLECTION_ID Requerida ID de la colección CMS de destino
DASHBOARD_PASSWORD Requerida Contraseña para acceder al dashboard
API_SECRET_KEY Auto Generada automáticamente por install.sh
MAX_PAGES Opcional Páginas a scrapear. 0 = ilimitado
MIN_SURFACE_M2 Opcional Superficie mínima en m². Por defecto: 1000
paso 3b — guardar sesión MilAnuncios (login manual) 
source venv/bin/activate && python save_session.py
Se abrirá Chrome. Inicia sesión en MilAnuncios y pulsa Enter en el terminal para guardar las cookies.
4
Arrancar API + Dashboard
El cron diario (6:00 Madrid) se activa automáticamente con la API
terminal — background, sobrevive al cierre del terminal 
bash start.sh
check_circle Dashboard: http://localhost:8501 — usa la contraseña de DASHBOARD_PASSWORD
API health: http://localhost:8000/health — debe devolver {"status":"ok"}
Cron automático
schedule
Programación automática del scraper
Se activa solo al arrancar la API — sin configuración extra
expand_more
check_circle Por defecto el scraper corre todos los días a las 6:00 (Europe/Madrid). El horario persiste aunque reinicies la API.

Ver el horario configurado:

GET /api/cron 
curl http://localhost:8000/api/cron \
  -H "x-api-key: TU_API_SECRET_KEY"

Cambiar el horario (ejemplo: 8:30):

PUT /api/cron 
curl -X PUT http://localhost:8000/api/cron \
  -H "x-api-key: TU_API_SECRET_KEY" \
  -H "Content-Type: application/json" \
  -d '{"hour": 8, "minute": 30}'

Lanzar el scraper manualmente ahora:

POST /api/scraper/run 
curl -X POST http://localhost:8000/api/scraper/run \
  -H "x-api-key: TU_API_SECRET_KEY"
Mantenimiento
refresh
Renovar sesión expirada
Cuando aparece SessionExpiredException en los logs
expand_more
info Las cookies de MilAnuncios duran ~30 min de inactividad. El scraper para automáticamente y reporta la excepción.
terminal 
source venv/bin/activate && python save_session.py

O desde la API sin abrir el terminal:

POST /api/session/renew 
curl -X POST http://localhost:8000/api/session/renew \
  -H "x-api-key: TU_API_SECRET_KEY"
stop_circle
Parar / reiniciar los servicios
Detener API y dashboard o reiniciarlos con un solo comando
expand_more

Detener:

terminal 
bash stop.sh

Reiniciar (detiene y vuelve a arrancar):

terminal 
bash restart.sh

Iniciar de nuevo tras haber parado:

terminal 
bash start.sh
