Setup & Producción

De cero a producción — configura el scraper, la API y el dashboard
con un solo comando en tu Mac o servidor Linux

Paso a paso — Mac
1
Prerequisitos
Homebrew + Python 3.11 o superior
ℹ️ Si ya tienes Python 3.11+ puedes saltar al paso 2. Comprueba con python3 --version.
Terminal
# Instalar Homebrew (si no lo tienes)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Instalar Python 3.12
brew install python@3.12

# Verificar
python3 --version
git --version
2
Clonar y ejecutar el instalador
Un solo comando configura el entorno, instala dependencias y genera credenciales
Terminal
# Clonar el repositorio
git clone <URL_DEL_REPOSITORIO>
cd milanuncions_scraper

# Setup en un comando — crea venv, instala deps, genera .env y API key
bash install.sh
✅ install.sh crea automáticamente venv/, instala todas las dependencias, copia .env.example → .env y genera un API_SECRET_KEY aleatorio seguro.
3
Configurar el archivo .env
Edita las credenciales necesarias
Terminal
nano .env    # o: open -e .env  /  code .env
VariableDescripciónTipo
WEBFLOW_TOKEN Token de API de Webflow (Settings → Integrations → API token) Obligatorio
WEBFLOW_COLLECTION_ID ID de la colección de Webflow donde se publican las naves Obligatorio
DASHBOARD_PASSWORD Contraseña para acceder al dashboard Streamlit Obligatorio
API_SECRET_KEY Token de acceso a la API REST (generado automáticamente por install.sh) Auto-generado
MIN_SURFACE_M2 Superficie mínima en m² para incluir un anuncio (por defecto: 1000) Opcional
MAX_PAGES Páginas máximas a scrapear por ejecución (0 = sin límite) Opcional
DOWNLOAD_IMAGES Descargar imágenes localmente antes de subir a Webflow (true/false) Opcional
4
Configurar sesión de MilAnuncios
Login manual una vez — las cookies duran ~30 min y se renuevan desde el dashboard
⚠️ Este paso abre Chrome visualmente para que puedas hacer login. Requiere pantalla — en Mac funciona directamente, sin configuración adicional.
Terminal
# Activar el entorno virtual (si no lo tienes activado)
source venv/bin/activate

# Abrir Chrome y esperar al login manual (máx. 10 min)
python save_session.py
ℹ️ El script espera a que inicies sesión en MilAnuncios y guarda las cookies en session.json automáticamente. Las cookies expiran en ~30 min — puedes renovarlas desde el dashboard sin salir del terminal.
5
Verificar que todo funciona
Tests rápidos antes de pasar a producción
Terminal
# Verificar imports principales
python -c "from scraper_engine import main; print('scraper_engine OK')"
python -c "from api.main import app; print('API OK')"
python -c "from db import init_db; init_db(); print('DB OK')"

# Scrape de prueba — 1 página, sin escribir en DB
python scraper_engine.py --pages 1 --dry-run
6
Producción — ejecutar en background
Servicios persistentes que sobreviven al cierre del terminal
Terminal — arrancar servicios
# API en background (puerto 8000)
nohup bash run_api.sh > logs/api.log 2>&1 &
echo $! > logs/api.pid
echo "API iniciada con PID $(cat logs/api.pid)"

# Dashboard en background (puerto 8501)
nohup bash run_dashboard.sh > logs/dashboard.log 2>&1 &
echo $! > logs/dashboard.pid
echo "Dashboard iniciado con PID $(cat logs/dashboard.pid)"
Terminal — ver logs en tiempo real
# Ver logs de la API
tail -f logs/api.log

# Ver logs del dashboard
tail -f logs/dashboard.log

# Ver logs del scraper (cuando está en ejecución)
tail -f scraper.log
Terminal — parar servicios
# SIGTERM — espera a que terminen limpiamente
kill $(cat logs/api.pid) $(cat logs/dashboard.pid)

# Verificar que han parado
ps aux | grep -E "uvicorn|streamlit"
✅ El cron se configura desde el dashboard (sección Programación) o via API: PUT /api/cron. Por defecto ejecuta a las 6:00 AM hora Madrid.
Paso a paso — Linux (Ubuntu/Debian)
1
Prerequisitos
Python 3.11+, Git, y Xvfb para Chrome headful en servidor sin pantalla
bash
sudo apt update
sudo apt install python3.12 python3.12-venv git xvfb -y

# Verificar
python3 --version
git --version
⚠️ En Linux sin pantalla física, run_api.sh ya configura DISPLAY=:99 y lanza Xvfb automáticamente para que Chrome pueda ejecutarse en modo headful (requerido para evitar detección anti-bot).
2
Clonar y ejecutar el instalador
Un solo comando configura todo
bash
git clone <URL_DEL_REPOSITORIO>
cd milanuncions_scraper

bash install.sh
3
Configurar el archivo .env
Rellena las credenciales obligatorias
bash
nano .env    # o: vim .env
Contenido .env
WEBFLOW_TOKEN=tu_token_de_webflow
WEBFLOW_COLLECTION_ID=id_de_tu_coleccion
DASHBOARD_PASSWORD=una_contrasena_segura

# API_SECRET_KEY ya fue generada por install.sh — no la cambies
4
Configurar sesión de MilAnuncios
En servidor Linux necesitas acceso VNC o X11 forwarding para el login visual
ℹ️ Si el servidor no tiene pantalla, conecta primero via VNC o usa ssh -X (X11 forwarding) para que Chrome pueda abrirse visualmente.
bash — con X11 forwarding
# Conectar con X11 forwarding
ssh -X usuario@servidor

# En el servidor, activar venv y lanzar sesión
cd milanuncions_scraper
source venv/bin/activate
DISPLAY=:0 python save_session.py
5
Producción — ejecutar en background
Con nohup los servicios sobreviven al cierre del terminal
bash — arrancar
# API en background (Xvfb integrado en run_api.sh)
nohup bash run_api.sh > logs/api.log 2>&1 &
echo $! > logs/api.pid

# Dashboard en background
nohup bash run_dashboard.sh > logs/dashboard.log 2>&1 &
echo $! > logs/dashboard.pid

echo "API PID: $(cat logs/api.pid)"
echo "Dashboard PID: $(cat logs/dashboard.pid)"
bash — ver logs
tail -f logs/api.log
tail -f logs/dashboard.log
bash — parar
kill $(cat logs/api.pid) $(cat logs/dashboard.pid)
✅ Los servicios están activos. El scraper se ejecuta automáticamente según el cron configurado en el dashboard (por defecto: 6:00 AM hora Madrid).
