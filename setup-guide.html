Naves Scraper / Setup

Guía de instalación

Configura el scraper de naves industriales en tu Mac en menos de 10 minutos.

laptop_mac macOS · Python 3.11+
Antes de empezar
  • check_circle macOS 12 Monterey o superior
  • check_circle Homebrew instalado — /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  • check_circle Cuenta en Webflow con acceso a la API (token + Collection ID)
  • check_circle Cuenta en MilAnuncios (para el login manual de sesión)
Pasos de instalación
1
Instalar Python 3.11+
Verificar o instalar via Homebrew
expand_more

Comprueba si ya tienes Python 3.11 o superior:

terminal
python3 --version

Si la versión es menor a 3.11, instala con Homebrew:

terminal
brew install python@3.12
2
Clonar el repositorio
Descarga el código fuente
expand_more
terminal
git clone <URL_DEL_REPO> naves-scraper
cd naves-scraper
3
Ejecutar el instalador
Un comando configura todo el entorno
expand_more
info El script es idempotente: puedes ejecutarlo varias veces sin problema. Crea el entorno virtual, instala dependencias, genera .env y crea los directorios necesarios.
terminal
bash install.sh

Al finalizar verás las instrucciones de los siguientes pasos y la opción de arrancar los servicios en segundo plano.

4
Configurar las variables de entorno
Editar .env con tus credenciales
expand_more
warning El archivo .env contiene secretos. Nunca lo subas a Git — ya está en .gitignore.
terminal
nano .env
Variable Tipo Descripción
WEBFLOW_TOKEN Requerida Token de API de Webflow (perfil → API Access)
WEBFLOW_COLLECTION_ID Requerida ID de la colección CMS de destino
DASHBOARD_PASSWORD Requerida Contraseña para acceder al dashboard Streamlit
API_SECRET_KEY Auto Generada automáticamente por install.sh
DB_PATH Opcional Ruta al archivo SQLite. Por defecto: naves.db
MAX_PAGES Opcional Límite de páginas a scrapear. 0 = ilimitado
MIN_SURFACE_M2 Opcional Filtro mínimo de superficie en m². Por defecto: 1000
DOWNLOAD_IMAGES Opcional Descargar imágenes localmente. Por defecto: true
5
Configurar la sesión de MilAnuncios
Login manual en Chrome (~2 min)
expand_more
info El scraper usa una sesión autenticada para evitar bloqueos. Las cookies duran ~30 minutos de inactividad; repite este paso si el scraper detecta sesión expirada.
terminal
source venv/bin/activate
python save_session.py

Se abrirá Chrome. Inicia sesión en MilAnuncios, luego vuelve al terminal y pulsa Enter para guardar las cookies.

warning No elimines la carpeta chrome_profile/ — acumula confianza anti-bot a lo largo del tiempo.
6
Arrancar los servicios
API en :8000 · Dashboard en :8501
expand_more

Arranque en primer plano (desarrollo):

terminal — dos pestañas
bash run_api.sh         # → http://localhost:8000
bash run_dashboard.sh   # → http://localhost:8501

Arranque en segundo plano (producción, sobrevive al cierre del terminal):

terminal
nohup bash run_api.sh > logs/api.log 2>&1 &
nohup bash run_dashboard.sh > logs/dashboard.log 2>&1 &
7
Verificar que todo funciona
Comprueba el health de la API y abre el dashboard
expand_more
terminal
curl http://localhost:8000/health

Respuesta esperada:

json
{"status": "ok"}
check_circle Abre http://localhost:8501 en tu navegador para acceder al dashboard. Usa la contraseña configurada en DASHBOARD_PASSWORD.
Mantenimiento
refresh
Renovar sesión expirada
Cuando el scraper reporta SessionExpiredException
expand_more
terminal
source venv/bin/activate && python save_session.py

O desde la API (si está corriendo):

terminal
curl -X POST http://localhost:8000/api/session/renew \
  -H "x-api-key: TU_API_SECRET_KEY"
stop_circle
Parar los servicios
Si arrancaste en background con nohup
expand_more
terminal
kill $(cat logs/api.pid) $(cat logs/dashboard.pid)
article
Ver logs en tiempo real
Monitorizar la actividad del scraper
expand_more
terminal
tail -f logs/api.log
tail -f scraper.log
