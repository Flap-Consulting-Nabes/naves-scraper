Naves Scraper / Setup

Guía de instalación

Configura el scraper de naves industriales en menos de 10 minutos. Al finalizar tendrás el dashboard activo y el cron diario funcionando.

laptop_macmacOS 12+
terminalUbuntu 22+
codePython 3.11+
javascriptNode.js 20+
webGoogle Chrome
Antes de empezar
  • check_circle Cuenta en Webflow con acceso a la API — necesitarás el token y el Collection ID
  • check_circle Cuenta en MilAnuncios — para el login manual de sesión (solo la primera vez)
  • check_circle Node.js 20+ instalado — node --version debe mostrar v20 o superior
  • check_circle Google Chrome instalado — el scraper usa Chrome con perfil persistente para evitar bloqueos anti-bot
  • check_circle Conexión a internet activa — el instalador descarga todo lo necesario
Instalación completa — 4 pasos
1
Instalar Homebrew, git y Python
macOS — ejecuta en orden, una sola vez
terminal — instalar Homebrew 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
terminal — instalar git y Python 3.12 
brew install git python@3.12
info Google Chrome debe estar instalado. Descarga desde google.com/chrome si no lo tienes. El scraper lo necesita para navegar MilAnuncios.
2
Clonar el repositorio e instalar
El instalador configura todo: venv, dependencias y .env
terminal 
git clone https://github.com/Flap-Consulting-Nabes/naves-scraper.git naves-scraper
cd naves-scraper
bash install.sh
Al terminar el script muestra el API_SECRET_KEY generado — guárdalo.
3
Configurar credenciales y sesión
Edita .env con tus claves y guarda las cookies de MilAnuncios
warning .env contiene secretos — nunca lo subas a Git. Ya está en .gitignore.

El instalador crea el .env con valores por defecto. Solo necesitas rellenar los tres campos marcados como Requerida.

paso 3a — abrir .env 
nano .env
VariableTipoDescripción
WEBFLOW_TOKEN Requerida Token de API de Webflow — perfil → API Access
WEBFLOW_COLLECTION_ID Requerida ID de la colección CMS de destino en Webflow
DASHBOARD_PASSWORD Requerida Contraseña de acceso al dashboard (http://localhost:3000)
API_SECRET_KEY Auto Generada por install.sh. Para regenerar: python -c "import secrets; print(secrets.token_hex(32))"
MAX_PAGES Opcional Páginas a scrapear por ejecución. 0 = ilimitado
MIN_SURFACE_M2 Opcional Superficie mínima de las naves a guardar. Por defecto: 1000
DOWNLOAD_IMAGES Opcional Descarga imágenes de los anuncios. true / false. Por defecto: true
IMAGES_DIR Opcional Carpeta local donde se guardan las imágenes. Por defecto: images
DB_PATH Opcional Ruta al archivo SQLite. Por defecto: naves.db
API_BASE_URL Opcional URL base de la API interna usada por el dashboard. Por defecto: http://localhost:8000
PROXY_SERVER Opcional Proxy HTTP para Chrome. Formato: http://user:pass@host:port. Desactivado si no se define.

Plantilla completa del .env para copiar y pegar:

.env — plantilla completa 
# ── Scraper ────────────────────────────────────────────────────────────────
DB_PATH=naves.db
MAX_PAGES=0
MIN_SURFACE_M2=1000
DOWNLOAD_IMAGES=true
IMAGES_DIR=images
# PROXY_SERVER=http://user:pass@host:port

# ── Webflow CMS ────────────────────────────────────────────────────────────
WEBFLOW_TOKEN=TU_TOKEN_WEBFLOW
WEBFLOW_COLLECTION_ID=TU_COLLECTION_ID

# ── API / Microservicio ────────────────────────────────────────────────────
# Generada automaticamente por install.sh — no editar manualmente
API_SECRET_KEY=

# ── Dashboard ─────────────────────────────────────────────────────────────
DASHBOARD_PASSWORD=cambia_esta_contrasena
API_BASE_URL=http://localhost:8000
paso 3b — guardar sesión MilAnuncios (login manual) 
source venv/bin/activate && python save_session.py
Se abrirá Chrome. Inicia sesión en MilAnuncios y navega a Mis Anuncios. El script detecta el login automáticamente y guarda las cookies — no es necesario pulsar nada.
4
Arrancar API + Dashboard
El cron diario (6:00 Madrid) se activa automáticamente con la API
terminal — background, sobrevive al cierre del terminal 
bash start.sh
check_circle Dashboard: http://localhost:3000 — usa la contraseña de DASHBOARD_PASSWORD
API health: http://localhost:8000/health — debe devolver {"status":"ok"}
